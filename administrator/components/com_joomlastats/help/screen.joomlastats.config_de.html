JoomlaStats ist eine Zusatzkomponente für das Contentmanagementsystem (CMS) Joomla! und bietet dem Webseitenbetrieber umfangreiche Statistiken für die aufgerufenen, eigenen Webseiten sowie zu Besuchern, Suchmaschinen, Bots, usw.

*Die Heimatseite von Joomlastats ist hier zu finden.
*Die jeweils aktuellste Version ist hier zu beziehen.
* Unterstützung gibt es auf JoomlaStats.org im Forum.
Hinweis: Um JoomlaStats vollständig in das CMS einzubinden, ist es erforderlich dass auch das Modul JooStats installiert und veröffentlicht wird!
Sollen alle Webseiten in die Statistik aufgenommen werden, sollte dieses Modul im ersten veröffentlichten Modul mit der Seitenzuweisung Alles eingebunden werden
JoomlaStats besteht im wesentlichen aus den folgenden Programmpunkten:
  • Statistik
    Von hier aus werden alle Statistiken aufgerufen
  • Konfiguration
    Hier werden alle Einstellungen zu JoomlaStats vorgenommen
  • Ausschluss
    IP-Adressen von der Statistik aus- oder wieder einschliessen
  • TDL-Check
    Kann ein Besucher automatisch keinem Land zugeordnet werden, kann hier ein Check dieser IP-Adressen durchgeführt werden
  • Info
    Informationen zu JoomlaStats und der verwendeten Datenbank
  • Hilfe
    Ruft die integrierte Hilfe auf (dieses Dokument hier)

  • Allgemein
    • Onlinezeit
      Für welchen Zeitraum (wählbar von 15 Minuten bis 24 Stunden) gilt ein Besucher als ein (1) Besucher
    • Startoption
      In welcher Ansicht soll die Statistik beim ersten Aufrufen gestartet werden
    • Sprachen
      Normalerweise kann JoomlaStats mit einer der Admin- bzw. Benutzersprache automatisch angepassten Sprache betrieben werden (Auto).
      Soll jedoch eine Sprache fix sein oder die automatische Einstellung überschrieben werden, kann hier aus den vorhandenen Sprachen eine gewählt werden
    • Zeitlimit
      Zeitlimit für interne Prozesse von JoomlaStats (z.B. die Zusammenfassung) - abhängig auch vond en vorhandenen Servereinstellungen.
      Lassen es die servervoreinstellungen zu, kann hier das Zeitlimit für z.B. Sicherungen verändert werden
    • Zusammengefasste Daten extra
      Wird die Datenbanktabelle mit den dynamischen Besucherdaten zusammengefasst, kann hier bestimtm werden dass diese ausgelagerten Daten extra angezeigt werden (markiert mit einen Stern *)
    • WHOIS Unterstützung
      Kann ein besucher keinem Land oder Domain zugeordnet werden, kann JoomlaStats automatisch versuchen diese Bestimmung (im Hintergrund) durchzuführen
    • I18n Unterstützung
      Wird das CMS mit einem Übersetzungstool wie z.B. Joomla!Fish betrieben (damit lassen die Artikel in der jeweiligen Besichersprache anzeigen), wird JoomlaStats diese Seiten als jeweils eine (1) Seite anzeigen
Seitenanfang
  • Tools
    • Datenbank zusammenfassen
      Mit dieser Aktion kann die bereits erwähnte Besucherdatenbank ausgelagert werden. Danach werden diese Daten mit einem Stern * gekennzeichnet. Diese Statistikdaten können dann teilweise nicht mehr einzeln abgerufen werden!
      Diese Aktion sollte spätestens dann durchgeführt werden, wenn die Datenbank zu gross wird (ersichtlich in JoomlaStats auf der Statistikseite in der ersten Zeile links).
      Hinweis: Diese Aktion ist erst dann sichtbar, wenn bereits Daten vorhanden sind!
    • Datenbank löschen
      Löscht unwiderruflich alle vorhandenen Daten ohne vorige Sicherung!
    • Teilsicherung
      Damit wird die Besucherstatistik gesichert. Danach wird diese Tabelle neu angelegt und steht ohne Inhalte wieder zur Verfügung
    • Komplettsicherung
      Es werden alle vorhandenen JoomlaStats-Tabellen gesichert und mit einem Zeitstempel (Datum und Zeit der Sicherung) gespeichert.
      Es kann auf diese Daten nur einem SQL.Manager (z.B. phpMyAdmin) zugegriffen werden!
Seitenanfang
  • Export
    • CSV Tabellenexport
      Mit dieser Aktion kann eine der ausgewählten Tabellen lokal als CSV (Comma Seperated Value) Datei gespeichert werden.
      Diese lokal gespeicherte Datei kann dann z.B. mit MS-Excel oder OpenOffice geöffnet und bearbeitet/angesehen werden
